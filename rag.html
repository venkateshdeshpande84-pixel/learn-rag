What each step means

1
Chunking

Split large documents into passages small enough to retrieve precisely.

2
Embedding

Convert the meaning of each passage into a numerical representation.

3
Vector store

Store each vector together with its original text and metadata.

4
Retrieval

Compare the query with indexed chunks and return the most relevant ones.

5
Reranking

Reassess the first results using a more precise relevance calculation.

6
Generation

Give the retrieved text to the LLM so it can answer with better context.

Important correction

Vectors are not converted back into language. The database keeps the original chunk text beside each vector. Retrieval uses vectors to locate records, then sends the original text chunks to the LLM.

Mail-person analogy

The vector is like a location code that helps narrow down the neighbourhood. The stored chunk is the actual address card. The user's query is a new address request. Retrieval finds the closest address cards. The prompt gives those cards to the LLM. The LLM reads the address cards and explains where to deliver the answer.
Educational simulation only — no external AI model or vector database is called.
